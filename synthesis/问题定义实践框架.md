# 问题定义实践框架

## Thesis

好的解题流程不是从答案开始，而是从定义问题开始：看见现象后，先确认差距、主人、重述方式和方案副作用，再进入方案设计。

## Framework

1. 看见现象：先描述现实，不急着归因。
2. 定义差距：现实状态与期望状态之间是否有明确落差。
3. 找问题主人：谁受影响、谁在意、谁有动力和能力推动改变。
4. 拆掉预设方案：把“要上系统/要降价/要机器人”重述成目标或待验证假设。
5. 重述问题：换主体、目标、尺度、时间和边界。
6. 评估副作用：方案会改变谁的行为，把成本转移给谁，短期长期是否一致。
7. 决定方案：在多个成立答案之间做取舍，而不是迷信第一个正确答案。

## Application To Agent Work

- 给 Agent 下任务前，先把“方案句”重述成“问题句”。
- 如果用户只给出第一个问题定义，先查现象、目标、问题主人和约束。
- HAT/验收标准应覆盖方案副作用，而不只覆盖主流程成功。
- 多 Agent 并发前，先确认每个 slice 的问题主人和完成证据，避免执行错方向。
