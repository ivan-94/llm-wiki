# 结构化文档解析

## Definition

结构化文档解析是把 PDF、图片、Office 文档等复杂资料解析成带布局、阅读顺序、表格、公式、图像和语义结构的中间表示，供检索、RAG 或 LLM 消费。

## Why It Matters

知识库质量常常卡在 ingest 前处理。只做 OCR 会丢失表格、版面、阅读顺序和图文关系；这些结构缺失会让 RAG 和总结结果失真。

## Mental Model

它不是“读文字”，而是“重建文档”：先识别页面上的对象，再恢复对象之间的顺序和结构。

## Key Claims

- 文档解析流水线通常包括布局识别、OCR/文本转换、表格识别、公式识别、图像裁剪和文档恢复。
- Document AI 需要结合文本、2D 位置、图像特征和布局信息。
- 面向 RAG 的解析应保留检索所需的块、表格、标题、阅读顺序和来源定位。
- 多格式资料预处理要先按格式转换，再归一成 Markdown/HTML/结构化文本层。
- 搜索/网页抓取也属于资料预处理链路，需要发现、抓取、正文抽取和结构化返回。

## Examples

- `结构化文档提取.xmind` 比较 LayoutLMv3、DiT、PaddleOCR PP-Structure、Table Transformer、RAGFlow DeepDoc 等路线。
- `资料预处理.xmind` 汇总 PDF、Word、Excel、图片和网页转换。
- `搜索.xmind` 给出搜索发现、页面抓取和正文抽取工具链。
- `音视频.xmind` 将音视频资料转换为字幕/文本作为另一类预处理。

## Common Confusions

- OCR 不是结构化文档解析的全部。
- PDF 渲染能看到页面，不等于提取了可用于 RAG 的结构。
- 表格识别不仅要识别文字，还要识别单元格结构和表格语义。
