# 预训练与微调范式

## Definition

预训练与微调范式是现代 LLM 的两段式训练框架：先在大规模无标注语料上做自监督预训练得到“会语言”的基础模型，再在较小的任务/偏好数据上微调，使模型适配具体任务、风格或人类偏好。

## Why It Matters

它是理解 LLM 训练管线的总纲：base model、SFT model、RM、RL model 这些阶段都落在这条轴上。先有这张全景图，才能把 SFT、RLHF、PEFT 等具体技术放到正确位置。

## Mental Model

通识教育 + 岗前培训：预训练像读完海量书形成通用语言能力，但还不“听话”；微调像针对岗位做培训，让模型按任务要求、风格和偏好工作。

## Key Claims

- 预训练在大规模无标签语料上以预测下一个 token 为目标，成本高，产出通用基础模型（explicit，`模型微调.xmind` / `State of GPT.pdf`）。
- 微调是在预训练模型上用特定任务数据继续训练，通常是有监督、数据集更小（explicit，`模型微调.xmind`）。
- 训练数据形态对应阶段：无标注数据→预训练，标注问答数据→微调，带正误偏好数据→对齐（explicit，`大模型.xmind`）。
- GPT Assistant 训练管线分四阶段：Pretraining → Supervised Finetuning → Reward Modeling → Reinforcement Learning（explicit，`State of GPT.pdf` 预览）。
- 预训练数据为公开来源混合，如 CommonCrawl、C4、GitHub、Wikipedia、Books、ArXiv、StackExchange（explicit，`State of GPT.pdf` 预览）。
- base 模型只会续写、不一定听指令，chat 能力来自后续指令/对齐训练（explicit，`大模型.xmind` / `微调生成模型.xmind`）。

## Common Confusions

- 微调不会重做预训练；它在已有权重上做较小学习率的继续训练，避免破坏通用能力。
- “阶段”（预训练/SFT/对齐）与“参数策略”（全参 vs PEFT/LoRA）是不同的轴，不要混为一谈。
- 注入新知识不一定靠微调，RAG/上下文工程往往更合适（见模型工程的上下文层 vs 参数层）。

## My Understanding

当前理解：先预训练得到“会语言但不听话”的基础模型，再分 SFT、对齐两步把它调成可用助手；这条轴是所有具体微调技术的坐标系。
