# 自回归语言模型

## Definition

自回归语言模型（autoregressive LM）是用过去的 token 预测下一个 token 的概率模型：它一次生成一个 token，把已生成内容重新作为上下文，循环预测直到结束。

## Why It Matters

GPT 系列、ChatGPT 以及大多数生成式 LLM 都是自回归模型。理解“逐 token、基于前文、概率预测”是理解解码策略、KV Cache、上下文窗口和推理成本的共同前提。

## Mental Model

像接龙：模型每次只押下一个字，押完把这个字接到句子末尾，再押下一个；输出不是一次算出整段，而是把同一个“预测下一个 token”的动作重复很多遍。

## Key Claims

- 语言模型一次生成一个 token，根据前面内容预测后面内容，这被称为自回归（explicit，`语言模型内部原理.xmind`）。
- 自回归思想与时间序列预测同源：都用过去观测值预测未来值（explicit，`机器学习/faq.xmind`）。
- LM Head 把隐藏状态映射到词表大小的输出空间，得到下一个 token 的概率分布（explicit，`语言模型内部原理.xmind`）。
- base 模型只是“预测后续文本”，并不一定是对指令或对话的响应（explicit，`大模型.xmind`）。
- 自回归生成天然串行，是 KV Cache 等推理优化的根本动因（inferred）。

## Examples

- `语言模型内部原理.xmind` 把外部表现（逐 token 生成）还原为内部 LM Head + 解码策略。
- `python 深度学习入门.xmind` 的语言模型 sheet 用“给定上下文预测下一个词元”定义语言模型，并用 Mini GPT 走完数据→分词→Decoder→采样推理。
- `图解ChatGPT.xmind` 用“逐词生成连贯回复”解释 ChatGPT 的外部行为。

## Common Confusions

- 自回归 ≠ 自编码：BERT 类 masked LM 不是逐 token 串行生成，而是双向重建被遮盖的 token。
- “预测下一个 token”是训练与推理的共同目标，但 base 模型会预测、不代表会“听话”，指令遵循来自后续微调。

## My Understanding

当前理解：自回归就是把“预测下一个 token”这一件事重复执行；它解释了为什么 LLM 的延迟随输出长度线性增长，以及为什么需要 KV Cache 和采样策略。
