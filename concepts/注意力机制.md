# 注意力机制

## Definition

注意力机制（attention）让模型在处理某个位置时，按相关性对其他位置的信息打分、加权并汇总，从而动态决定“此刻该关注哪些 token”。

## Why It Matters

注意力是 Transformer 的核心构件，也是 LLM 能建模长距离依赖、并行处理序列的根本原因。理解 query/key/value 的点积加权，是看懂 Transformer、KV Cache 和多头机制的钥匙。

## Mental Model

检索类比：当前 token 拿着一个 query（“我要找什么信息”），去和每个 token 的 key（“我能提供什么”）做匹配打分，softmax 归一化成权重，再对各 token 的 value（“具体信息”）加权求和，得到融合了上下文的新表示。

## Key Claims

- 注意力通过打分、softmax 权重、加权求和选择关键输入；自注意力让序列内部位置相互关联（explicit，`Transformer.xmind`）。
- query、key、value 是三种投影矩阵，通过线性变换帮助计算注意力分数与加权求和（explicit，`语言模型内部原理.xmind`）。
- query 与 key 的点积产生相关性分数，softmax 归一化后对 value 加权求和（explicit，`语言模型内部原理.xmind`）。
- 多头注意力并行复制注意力，让模型同时从不同表示空间关注不同模式（explicit，`语言模型内部原理.xmind` / `Transformer.xmind`）。
- 注意力最初是为缓解 RNN 长序列信息衰减而引入，使模型能直接访问过去隐藏状态（explicit，`Token and Embedding.xmind`）。
- 因果掩码（causal mask）使解码器只能看到当前位置及之前的 token（explicit，`python 深度学习入门.xmind`）。

## Examples

- `语言模型内部原理.xmind` 给出 query/key/value 三投影 + 点积 + softmax + 加权求和的完整链路。
- `Transformer.xmind` 把注意力放在 RNN→注意力→自注意力→多头注意力的演进台阶上。
- `python 深度学习入门.xmind` 的 Transformer sheet 列出多头注意力、位置编码、因果掩码等组件。

## Common Confusions

- 自注意力（序列内部相互关注）与多头注意力（多组并行注意力）不是同一层概念，后者是前者的并行扩展。
- query/key/value 不是天生存在的，而是同一输入经三组不同投影矩阵线性变换得到。
- 因果掩码只在自回归/解码场景需要；encoder 的双向注意力不加因果掩码。

## My Understanding

当前理解：注意力把“关注谁”变成可学习的加权求和。query 找信息、key 被匹配、value 被聚合，softmax 决定权重；多头让模型同时看多种关系。
