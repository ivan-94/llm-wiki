# 解码与采样策略

## Definition

解码与采样策略决定如何从语言模型每一步输出的 token 概率分布中选出下一个 token：可以贪婪取最高概率，也可以按温度等参数引入随机性。

## Why It Matters

同一个模型，解码策略不同，输出的确定性、多样性和“有趣程度”差别很大。它是推理时无需重新训练就能调节输出风格的旋钮，直接影响生成质量与可复现性。

## Mental Model

模型每步给出一张“候选词概率表”，解码策略是“怎么从这张表里挑”：要稳就挑最高分（贪婪），要活就按概率掷骰子，温度则控制骰子的“随机强度”。

## Key Claims

- 解码策略决定如何从概率列表中选择结果（explicit，`语言模型内部原理.xmind`）。
- greedy decoding 选概率最高的 token，最简单但通常较无趣（explicit，`语言模型内部原理.xmind`）。
- 推理时可使用贪婪采样或温度采样控制输出；温度影响输出随机性（explicit，`python 深度学习入门.xmind` / `图解ChatGPT.xmind`）。
- LLM 生成循环为：分词 → embedding → Transformer 层 → logits → 采样 → 重复直到目标长度或 EOS（explicit，`大模型.xmind`）。
- 温度越高输出越随机多样，越低越确定保守（inferred，基于 source 对温度作用的描述）。

## Examples

- `语言模型内部原理.xmind` 用 greedy decoding 作为最简策略基线。
- `图解ChatGPT.xmind` 指出生成时可从多个候选词中选择，温度影响随机性。
- `大模型.xmind` 把采样作为推理流程中 logits 之后、EOS 之前的一步。

## Common Confusions

- 采样策略是推理时的选择规则，不改变模型权重，与微调无关。
- 温度不是“质量旋钮”：高温更有创意但更易跑偏，低温更稳但更易重复无趣。
- EOS（结束符）由模型预测，是停止生成的信号之一，与最大长度限制是两回事。

## My Understanding

当前理解：解码策略是“从概率表里怎么挑下一个 token”。贪婪最稳但无趣，温度采样在确定性和多样性之间调节，是免训练的输出调参手段。
