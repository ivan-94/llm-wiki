# 工程宪法

## Definition

工程宪法是项目级、长期稳定的实现原则和禁止反模式，用来约束 Agent 在具体 feature spec 未覆盖的工程判断中保持一致。

## Why It Matters

Feature spec 无法重复写完所有工程价值函数。工程宪法把复用抽象、规则归位、单一事实源、命名、依赖和重构边界前置为全局约束。

## Mental Model

工程宪法是 Agent 写代码时不能静默违背的底层法律。

## Key Claims

- 工程宪法应从 feature spec 中抽离，保持跨需求稳定。
- 它应定义优先复用、业务规则归位、单一事实源、最小可维护实现和变化点开放。
- 它应禁止测试特判、复制业务规则、绕过 service、平行类型、broad catch 掩盖错误、前端成为唯一安全边界等反模式。
