# 问题重述

## Definition

问题重述不是换词，而是重新组织问题的边界、视角与重点，从而改变分析路径和方案空间。它是 [[concepts/问题定义|问题定义]] 系列第 5 篇主题，被称为“解决问题的核心动作”。

## Why It Matters

同一个问题换一种表述，可能带来完全不同的解决方向。重述能防止被表面说法带偏、看清真正目标、打开更多解法、发现边界与取舍。先重述再评估方案，通常比直接解决更有效。

## Mental Model

重述流程：听到原始表述 → 追问现象与目标 → 换角度重述 → 看到新结构 → 找到新解法。

五个重述方向：换主体、换目标、换尺度、换时间、换边界。

重述类型转换：从现象→差距，从归因→待验证问题，从方案→目标，从情绪→结构。

## Key Claims

- 重述不是文字游戏，而是重新组织边界、视角与重点。
- 同一问题换表述会带来不同分析路径和方向。
- 重述可从主体、目标、尺度、时间、边界入手。
- 常见误区：只换词不换理解、越重述越模糊、把重述当结论、只从自己角度重述。

## Examples

- “客户总说我们服务差” → “客户等待时间太长” → “高峰期接待流程让客户等待超出预期” → 导向“优化分流、预约、前台协作，而不只是多招人”。
- 把“要不要上系统”重述为“信息为什么传不动”。

## Common Confusions

- 以为重述就是改措辞——核心是换理解角度。
- 把一次重述当最终结论——重述是持续校准。
