# 系统提示语

## Definition

系统提示语是比单次用户请求更高优先级、更长期有效的行为契约，用来规定助手身份、能力边界、输出习惯、安全约束和失败处理方式。

## Why It Matters

普通 prompt 解决某次任务怎么做；系统提示语决定模型在一类任务和多轮交互中应该保持什么行为。它影响模型的默认语气、边界判断、工具使用预期和风险处理。

## Mental Model

把系统提示语看成 LLM 应用的“运行时政策”：它不是每次都直接回答用户问题，但会约束每次回答的身份、边界和风格。

## Key Claims

- 成熟系统提示语通常先说明身份、当前日期和知识边界，再说明能力范围、不能做的事和回答习惯。
- 能力边界要写清楚，例如无法打开链接时应要求用户粘贴内容，而不是假装已访问。
- 失败处理也是系统提示语的一部分，例如不能执行任务时直接说明，不用道歉填充。
- 多模态系统提示语需要处理图像内指令、隐私和识别边界。
- 系统提示语不只是安全规则，也承载产品体验、输出格式和任务分段策略。

## Examples

- Claude 系统提示语案例中包含日期、知识库更新时间、能力范围、复杂问题详答、简单问题简答、代码使用 Markdown 等规则。
- 图像相关系统提示语要求面对包含人脸的图片时不识别具体个人，并先复述图中指令。
- 长任务规则可以要求模型分段完成，并在每部分结束后寻求反馈。

## Common Confusions

- 系统提示语不是越长越好；冗余规则会增加上下文负担，也可能互相冲突。
- 系统提示语不能替代评估；写了规则仍需要用数据集和真实场景验证遵循度。
- 产品信息和模型家族信息容易过期，不能把历史系统提示语当作当前事实来源。
