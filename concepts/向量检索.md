# 向量检索

## Definition

向量检索是把文档和查询转换为高维向量表示（embedding），再通过近似最近邻搜索（ANN）找到语义相近内容的检索方式。它是 RAG 系统的核心检索机制之一。

## Why It Matters

传统关键词搜索无法处理语义相似但词语不同的情况，例如"最大规模"和"capacity 上限"在语义上接近但关键词不匹配。向量检索通过统一语义空间解决这一问题，是现代 RAG 系统的基础组件。

## Mental Model

向量检索是在"语义地图"上找最近邻：每个文档和每个查询都是这张地图上的点，检索就是找离查询点最近的文档点，不管它们用了什么词。

## Key Claims

- explicit（来自 Langchain 组件.xmind）：LangChain 的 Retrieval 分支把 embedding、向量数据库和检索器作为 RAG 的核心组件链。
- explicit（来自 RAG.xmind）：混合检索把向量搜索与关键词全文搜索组合，以弥补各自局限。
- inferred：向量检索的质量上限由 embedding 模型的质量决定；embedding 模型的上下文窗口也限制了可处理的文档粒度。
- explicit（来自 Langchain 组件.xmind）：LangChain 提供 Vectorstore、ParentDocument、Multi Vector、Self Query、Contextual Compression、Time-Weighted、Multi-Query、Ensemble、Long-Context Reorder 等检索器策略。

## Examples

- 用 OpenAIEmbeddings 把文档块 embedding 后存入 Chroma，再对用户查询做同样 embedding 后检索 top-k。
- Multi-Query Retriever：对同一问题自动生成多个改写版本，分别检索后合并，覆盖表达差异。
- Self Query Retriever：把查询和过滤条件分开，query 部分做向量搜索，filter 部分做元数据过滤，两者结合召回。

## Common Confusions

- 向量检索和关键词搜索各有盲区，单独使用时均有缺陷；生产场景推荐混合检索。
- ANN（近似最近邻）是近似搜索，可能错过严格最近邻文档，但实际精度通常足够。
- embedding 模型的语言和领域偏差会影响检索质量；跨领域或多语言场景需要验证 embedding 效果。

## My Understanding

向量检索解决了"语义匹配"问题，但存在精确术语召回弱的缺口。它应该作为混合检索体系的一部分，而不是单独依赖。
