# 行为契约式规格

## Definition

行为契约式规格是用可观察行为、requirement、scenario、Given/When/Then 和反例来描述系统必须表现出的外部契约，而不是直接规定内部实现。

## Why It Matters

Agent 容易在未定义空间里补全实现。行为契约把“什么算正确”变成可检查对象，降低需求偏移和测试过拟合。

## Mental Model

行为契约是系统对外承诺的 testable language。

## Key Claims

- OpenSpec 把 spec 视为 observable behavior contract，而不是实现细节。
- 每个 requirement 至少需要 scenario，MODIFIED 要保留完整 block，才能防止 delta 语义丢失。
- SbE 用具体例子补强行为边界，适合作为行为契约的实践方法。
- 行为契约应连接验收、测试和归档，而不只是文档。
