# Agentic Software

## Definition

Agentic Software 是以目标、上下文、记忆和编排为核心的软件形态：用户表达意图，系统拆解任务、选择能力、生成或呈现合适界面，并在执行中保留可检查、可回滚、可验证的控制点。

## Why It Matters

它把软件竞争点从“功能列表”推向“上下文资产 + 可控执行”。这能解释为什么传统 SaaS、AI Agent、动态 UI、AI 眼镜和原子能力编排会被放在同一个问题框架里。

## Mental Model

传统软件像固定菜单；Agentic Software 像一个有记忆、有工具、有审计轨迹的任务编排层。

## Key Claims

- Agentic Software 不是给传统软件加聊天框，而是重排入口、资产、架构和商业模式。
- UI 会从固定界面转向场景化、临时性、个体化界面。
- 记忆、偏好、历史行为、约定和 SOP 会成为软件壁垒。
- 可控性必须内建，包括计划可见、中间结果可检查、关键步骤确认、过程可回滚和逐步验证。

## My Understanding

当前理解：Agentic Software 的核心不是 Agent 本身，而是让“目标到结果”的软件过程可以个性化、可审计、可恢复。
