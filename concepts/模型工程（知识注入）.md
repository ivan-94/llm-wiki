# 模型工程（知识注入）

## Definition

模型工程把“给模型注入新知识/能力”的方式分成两层：上下文层（提示语工程，通过上下文学习把示例和任务知识放进输入）和参数层（模型工程，通过训练/微调更新模型权重）。模型工程特指后者——把知识固化进参数。

## Why It Matters

它提供一个统一框架回答“该改提示还是该微调”：上下文层轻、快、易迭代，参数层重但能把稳定领域知识、风格和能力固化。这是 LLM 应用优化选型的上位判断。

## Mental Model

便利贴 vs 写进脑子：提示语工程像往输入里贴便利贴，模型当场看一眼就忘；模型工程像把知识写进长期记忆（权重），一次训练长期生效但成本高。

## Key Claims

- 提示语工程和模型工程本质上都可理解为知识注入（explicit，`模型工程.xmind`）。
- 提示语工程通过上下文学习，借少量示例向模型注入新知识（explicit，`模型工程.xmind`）。
- 模型工程通过参数学习，即更新模型参数来学习新知识（explicit，`模型工程.xmind`）。
- 提高 LLM 性能可从提示词工程、微调、上下文学习几个方向展开（explicit，`提高 llm 性能.xmind`）。
- 选型分流：领域知识不在预训练语料可继续训练；有标注数据和任务范式可微调；回答须严格基于已有知识更适合 RAG 或微调结合 RAG（explicit，`大模型.xmind`）。

## Examples

- `模型工程.xmind` 用“知识注入”作为提示语工程与模型工程的共同上位概念。
- `大模型.xmind` 给出从零训练 / 继续训练 / 微调 / RAG / 对齐的场景分流。
- `提高 llm 性能.xmind` 把性能提升粗分为提示词工程、微调、上下文学习。

## Common Confusions

- 上下文层（提示/few-shot/RAG）不改权重，参数层（微调）改权重，二者是互补而非替代。
- “注入知识”不必默认微调；知识型、时效性任务常更适合 RAG。
- 模型工程是参数层优化的统称，微调/对齐/PEFT 都落在这一层。

## My Understanding

当前理解：所有“让模型懂新东西”的办法可分两层——往输入贴上下文（提示/RAG，便宜灵活）或改权重（微调，贵但固化）。模型工程是后者，选型先看知识是否稳定、是否在预训练语料里。
