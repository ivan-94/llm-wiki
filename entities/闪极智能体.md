# 闪极智能体

## What It Is

闪极智能体是一个以 AI 眼镜为核心入口的产品/项目，把眼镜语音、多模态环境捕获、全天录音、闪记/日程等自有应用，组合成一个可扩展的 Agent runtime，而不是继续维护多个互相隔离的单轮智能体。它与 sharge `ai_glass` 同属一条 AI 眼镜产品线的架构演进（早期 `ai_glass` 已有专家路由和记忆管家雏形）。

## Role In This Wiki

它是本 wiki 里「真实业务 Agent runtime 产品化」的主案例：从设备消息模型、MessageBus、运行时选型、会话存储压缩、多渠道记忆到 nsjail 沙箱，几乎覆盖 Agent 系统工程的每一层，可用来对照 [[concepts/Agent Runtime|Agent Runtime]]、[[concepts/Agent MessageBus|Agent MessageBus]]、[[concepts/会话分层存储与压缩|会话分层存储与压缩]]、[[concepts/多渠道记忆协议|多渠道记忆协议]] 和 [[concepts/Agent 沙箱|Agent 沙箱]]。

## Key Facts

- 架构分层：设备交互层、MessageBus、智能体运行时、LLM 网关、应用层、Channel；目标是把眼镜从「调用某个固定智能体」变成「承载可扩展 Agent runtime 的交互 Channel」。
- 消息模型：主动消息（唤醒词/按键，需短连、多轮、多模态、拍照/短视频、设备控制、HITL、WebSocket 保持）与被动消息（长连/轮询，用于通知、任务回调、危险操作确认）。
- 路由角色：名称路由、场景路由、命令匹配、复杂度计算分发到日程、搜索员、录音专家、记录员、记忆管家、营养师、规划大师等自定义 Agent（excalidraw 架构图），与 `ai_glass` 的 `match_expert_name`/`match_expert_expertise`/`ExpertClient` 一脉相承。
- Gateway / LLM 网关：负责标准 OpenAI-compatible API、调用跟踪、成本统计、多供应商后备；`/v1/chat/completions` 为最小集成面，通过 SSE `custom_event` 承载工具调用、reasoning、工具结果、结构化输出、中断和设备控制。
- 会话存储：独立 SQLite（`sessions`/`messages`/`messages_fts`/`schema_version`），按 turn 写入；压缩用 `messages`/`messages_staged` 分层 + Protect Head/Tail + Summary Area。
- Loomos 记忆：多渠道记忆协议（search/readMore/installTools），动态记忆=对话，静态记忆=闪记/相册/日程/录音。
- nsjail 沙箱：按 `user_id` 创建用户级 `NsJailService`，`bash -lc` 执行、SSE 流式输出、SQLite 记录命令元数据；单用户最多 5 并发、workspace 2GB 配额、单命令 1core/512MB/默认 30 分钟，明确「不追求强安全对抗」。
- MVP 取舍：单用户单会话、单应答串行、有限工具、闪极业务耦合；技术栈倾向 TypeScript、bun、Drizzle ORM、SQLite、Docker/dev container、nsjail、pi-mono agent core。
- 当前为内部产品/架构草案，外部链接与产品状态未联网核验。
