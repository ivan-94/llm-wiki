# Andrej Karpathy

## What It Is

Andrej Karpathy 是前 OpenAI 创始成员、特斯拉 AI 总监，以对 LLM、神经网络和 AI Coding 的公开观察和概念命名广为人知。他提出了 "Vibe Coding"（2025 年初）和 "Software 3.0" 等在 AI 工程社区影响广泛的术语，并在 X（Twitter）持续记录对 LLM 能力边界和工程实践的第一手观察。

## Role In This Wiki

Karpathy 是本 wiki Vibe Coding / Agentic Engineering 主题的重要思想来源，提供了：
- **Vibe Coding 定名和定义**：用自然语言让 AI 写代码，不关注实现细节的轻量开发方式。
- **Software 1.0/2.0/3.0 框架**：把编程范式从显式规则（1.0）到 ML 权重（2.0）再到 LLM 上下文程序（3.0）分层。
- **LLM 能力边界观察**：强项（代码、数学、可验证任务）vs. 弱项（常识、品味、训练分布外问题）。
- **Agent coding 行为规范**：认知纪律、简洁纪律和修改边界纪律。
- **LLM Wiki 模式**：把长期资料转化为 LLM 维护的结构化知识库。

## Key Facts

- 提出 "Vibe Coding" 一词，时间约为 2025 年初（X 帖子）。
- "Software 3.0" 把 LLM 定义为可通过 prompt、context、工具和环境来编程的计算机。
- 观察 LLM coding 行为时指出：模型容易把假设、复杂度和无关改动写进代码。
- 提倡 LLM Wiki 模式：raw 层 + wiki 层 + rules 层，知识通过持续编译而非孤立摘要积累。
- 当前与 OpenAI 创始团队的关联：原创始成员之一，后离开并加入特斯拉，再后来创办 AI 教育项目。
