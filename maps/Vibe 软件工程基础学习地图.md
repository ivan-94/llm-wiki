## Learning Path

1. 先读 [[concepts/Specification by Example|Specification by Example]]，理解示例如何变成规格和验收测试（含五质量门）。
2. 顺序阅读五张 SbE 图片：
   - [[sources/Vibe/软件工程基础/Specification by Example/1为什么需要 sbE.png|1 为什么需要 SbE]]：需求缺陷和返工成本。
   - [[sources/Vibe/软件工程基础/Specification by Example/2 sbE 是如何运作的.png|2 SbE 如何运作]]：业务规则→示例→规格→自动化测试。
   - [[sources/Vibe/软件工程基础/Specification by Example/3 团队如何做.png|3 团队如何做]]：Three Amigos 协作，Example Mapping。
   - [[sources/Vibe/软件工程基础/Specification by Example/4 组织和协作.png|4 组织和协作]]：规模化协作模式。
   - [[sources/Vibe/软件工程基础/Specification by Example/5 落地和反模式.png|5 落地和反模式]]：测试分层，活文档，常见反模式。
3. 读 [[sources/Vibe/软件工程基础/Specification by Example/sbE与DDD_BDD_TDD.png|sbE与DDD_BDD_TDD]]，理解 DDD、BDD、TDD 与 SbE 的协作链路；再深化读 [[synthesis/DDD–SbE–BDD–TDD 协作链路|DDD–SbE–BDD–TDD 协作链路]]。
4. 读 Cucumber/Gherkin 相关图：
   - [[sources/Vibe/软件工程基础/Specification by Example/cucumber-1.png|cucumber-1]]：Gherkin 场景结构。
   - [[sources/Vibe/软件工程基础/Specification by Example/cucumber-2.png|cucumber-2]]：自动化测试分层实现。
   - cucumber-3/4/5 原图尚未 ingest（raw 缺失或未同步），待补后再链入。
5. 读 [[sources/Vibe/软件工程基础/Specification by Example/示例.png|示例.png]]：SbE 具体示例格式参考。
6. 读 [[sources/Vibe/软件工程基础/Specification by Example/ai_coding.png|ai_coding.png]]：SbE 映射到 AI Coding 验收链路，理解 Agent DoD。
7. 回到 [[concepts/Spec-Driven Development|Spec-Driven Development]]，把示例驱动和规格驱动结合起来；读 [[synthesis/SbE 与 AI Coding 验收链路|SbE 与 AI Coding 验收链路]] 综合理解。
