# 提示语与上下文工程学习地图

## Purpose

组织提示语工程、上下文工程和 LLM 评估的学习路径，把"写 prompt"扩展为"设计任务接口、管理上下文、用评估验证效果"。

## Entry Points

- [[sources/提示语工程:上下文工程/提示语工程.xmind|提示语工程.xmind]]：学习目录入口。
- [[sources/提示语工程:上下文工程/上下文工程.xmind|上下文工程.xmind]]：核心理论入口。
- [[sources/提示语工程:上下文工程/案例.xmind|案例.xmind]]：系统提示语案例入口。
- [[sources/提示语工程:上下文工程/LLM 评估.xmind|LLM 评估.xmind]]：评估闭环入口。

## Learning Path

1. 先读 [[concepts/提示语工程|提示语工程]]，建立 prompt 是任务接口的理解。
2. 读 [[concepts/系统提示语|系统提示语]]，理解长期行为契约如何写入 prompt。
3. 回到 [[sources/提示语工程:上下文工程/提示语工程.xmind|提示语工程.xmind]] 的多 sheet 内容，按基础、通用技巧、高级技巧、故障排除、Agent 模式、长上下文模型、12 法则复习。
4. 再读 [[concepts/上下文工程|上下文工程]]，理解长上下文和 Agent 的失效模式。
5. 读 [[concepts/上下文失效模式|上下文失效模式]] 和 [[concepts/JIT 检索（Just-in-Time Retrieval）|JIT 检索]]，从失效模式入手，理解主动上下文管理的必要性。
6. 读 [[synthesis/提示语工程与上下文工程|提示语工程与上下文工程]]，区分两者边界和交叉点。
7. 读 [[concepts/LLM 评估|LLM 评估]] 和 [[concepts/LLM-as-a-Judge|LLM-as-a-Judge]]，把优化变成可验证实验。
8. 通过 [[entities/LangSmith|LangSmith]]（legacy · not verified）了解 LLM DevOps 平台如何把 trace、dataset、evaluator 和 CI 串成评估闭环，形成系统化评估视角。
9. 回到 [[synthesis/公文提示词优化评估清单|公文提示词优化评估清单]]，尝试把通用方法应用到垂直场景。
