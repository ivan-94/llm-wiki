## 分析与洞察

本周最稳定的工作方式变化，是从“Agent 读代码后给结论”转向“Agent 必须证明真实路径”。`sharge` 的 AI Daily waiting 问题不是接口不支持，而是默认时间窗口过滤了今天稍后的 waiting daily；`xmind-cli validate` 能过但 XMind App 打不开；Redis stats 的单测能过但真实 HAT 暴露事务刷新时机；Homebrew 发布只有经过 `brew install`、`brew test` 和 strict audit 才算用户路径闭环。这些事实指向同一个机制：Agent 容易把可推理的局部正确当成系统正确，而真实消费者路径会暴露环境、认证、旧数据、事务、工具宿主和 UI 语义中的隐藏假设。下周遇到状态列表、文件格式、缓存、发布渠道或客户端可见改动时，应把真实路径 smoke test 前移到设计或 HAT 准备阶段。

多 agent 并行本周带来了明显吞吐收益，也暴露了集成风险。`xmind-cli` 多 slice、`sharge` PRD 交付、AI Wiki `生图`/`Vibe`/全量 raw ingest、source link repair、7 天日报生成都依赖 worker 分片；成功的共同点是 worker 有清晰 write scope，父 Agent 负责合并、compile、HAT、diff、review 和最终状态判断。失败或摩擦也集中在同一层：shared implementation notes 回退其他 slice、worker patch 落到原 checkout、worker 没 final message、日报 worker 不能看到公开 thread 工具、线程上限导致第 7 个 worker 失败。并行不是把责任平均分给 worker，而是把局部执行并行化，同时把产品语义和交付收口集中到父 Agent。

AI Wiki 的建设显示出另一个转折：知识库质量不是 raw coverage，而是证据、关系和复用路径。早期重点是把 raw 资料镜像到 source note；后续用户纠偏指出抽象浅、实体稀疏、忽略 raw 目录关系、Source Links 像候选清单。于是 compile gate 开始要求 source cluster、typed relation、concept/entity/synthesis/map 和目录语义。这个变化很关键：如果只追求 `raw diff` 清零，wiki 会成为更整齐的资料堆；只有把 source-source、source-concept、source-entity 的关系写清楚，它才会成为可学习、可复习、可由后续 agent 继续编译的知识图谱。

本周还有一个反复出现的边界问题：规则、模板、skill、helper 和持久化产物各自应该承担不同职责。AGENTS 里不应堆类型细节，PDF preview 的 `partial` 语义不应被 helper 默认推成全局设计，Source Manifest 不应被旧示例反向污染，日报脚本不应越权生成正文，human/raw 不应被自动 ingest。每次用户纠偏都在压同一个倾向：Agent 喜欢把“为了完成当前任务的局部实现”上升成规则，或者把“脚本能生成的文本”当成最终判断。下周凡是修改规则或产物模板，都应该先问这是全局边界、类型策略、命令实现，还是一次性执行记录。

生成式工具和自动化工具本周都证明了“可生成”和“可信任”之间的距离。imagegen 能产出本地化商品图，但可能改屏幕读数、规格表和品牌细节；hooks 脚本模拟通过，不等于 Codex/Gemini 宿主已加载；PDF preview 可帮助 ingest，但不是全文 OCR；XMind Markdown export 少 hyperlink 会直接削弱 source evidence。工具链不是外挂，而是知识库和交付流程的一部分。下周要把工具能力缺口优先修在源头，而不是在 wiki 或报告层用文字补救。

## 启发与下周调整

- 将“真实路径验证”设为默认触发，而不是最后补测。触发条件是状态列表、发布安装、外部 App、浏览器移动端、缓存/统计、scheduler/hook 或商品图输出；工作方式是先定义一个最小真实消费者路径，再决定单测和脚本验证补在哪里。
- 多 worker 任务先写父 Agent 控制面。触发条件是同一任务要拆 3 个以上 worker，或多个 worker 可能碰共享文档、index/log、compile 页面、HAT guide；工作方式是先列 write scope、禁止范围、合并责任、worker final message 要求和父级回归命令。
- AI Wiki ingest 继续从 coverage 导向转为 graph 导向。触发条件是同一目录超过 10 个 raw、同一主题已有 3 个以上 source、或 source note Links 出现 candidate-only/plain text；工作方式是 raw diff preflight 后 worker 只写 source note，父 Agent 统一 compile concept/entity/synthesis/map 和 source-source typed links。
- 规则修改前做“归属判定”。触发条件是要编辑 AGENTS、模板、skill、helper 或日报/周报脚本；工作方式是先判断规则应处于全局硬边界、类型专属流程、命令实现还是产物写作规范，避免同一行为散落三处。
- 对生成式视觉和 preview ingest 保留人工/视觉验收口。触发条件是 imagegen 商品图、PDF preview、图片 source note 或包含屏幕读数/品牌/规格的素材；工作方式是脚本只验证尺寸、metadata、文件存在，最终合格必须明确人工或视觉模型检查范围。
