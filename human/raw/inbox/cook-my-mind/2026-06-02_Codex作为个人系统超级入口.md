# Codex 作为个人系统超级入口

## 我的原始想法

我觉得 Codex 已经成为了一个超级入口。我的所有的一切都围绕着他重新展开。

工作编码：Codex 主要用法

知识库：我依旧会使用手动创建 XMind 来整理我的单点知识，这种拆解、合并、链接的方式很适合我来消化和整理知识，所以我会坚持在某些方面手动 XMind。近期我开始基于 Codex 和 Obsidian 来重构知识库，也就是你现在看到的这个仓库。Obsidian 可以让我的单点知识在更高的维度串联起来，变成一个知识库的前端，而 Codex 是背后的大脑。

知识的捕获：我构建了 cook 系列的 skills，这些 skills 让外部的信息源在我开始 digest 之前先烹饪好，之前需要我自己 cook。

知识的闭环：cook digest ingest query cook digest ingest....

这一切都围绕着 Codex 这个 agent 展开。

## 速读

你正在把 Codex 从“写代码的工具”升级成个人数字系统的统一入口。编码、知识捕获、知识消化、Obsidian 知识库、XMind 手动拆解，都开始围绕 Codex 重新排布。

Agent 归纳：这不是单个工具偏好的变化，而是工作方式的重心迁移。Codex 负责把多个工具和流程串起来；XMind 保留手动深加工的触感；Obsidian 成为可浏览、可回顾、可互链的知识前端；cook skills 把外部信息预处理成可 digest 的输入。

## 我真正关心的问题

Agent 归纳：你真正关心的不是“用不用 Codex”，而是如何把 Codex 变成一个长期稳定的个人操作层，让工作编码、知识捕获、知识整理、知识复用进入同一个闭环。

这个问题可以拆成三层：

- 工具层：Codex、XMind、Obsidian、skills 各自承担什么角色。
- 流程层：cook、digest、ingest、query 如何循环。
- 认知层：哪些知识仍然需要你亲手拆解，哪些可以交给 Agent 预处理和连接。

## 想法地图

- 中心：Codex 是 agent 入口，也是背后的编排大脑。
- 工作编码：Codex 仍然是主要编码入口，从想法推进到可运行结果。
- 单点知识：XMind 继续负责局部主题的手动拆解、合并和链接。
- 知识前端：Obsidian 承接更高维度的连接、回顾和图谱化。
- 捕获前处理：cook skills 负责把外部信息源先处理成更适合 digest 的形态。
- 闭环：cook -> digest -> ingest -> query -> 新问题或新捕获 -> cook。

## 已经明确的判断

- Codex 已经不只是一个编码辅助，而是你的工作与知识系统的超级入口。
- XMind 不会被 Obsidian 或 Codex 替代；它保留在“单点知识消化”的位置。
- Obsidian 的价值在于把单点知识上升到更高维的连接层，成为知识库前端。
- cook 系列 skills 的价值在于降低外部信息进入 digest 前的摩擦。
- 你的知识工作正在形成一个闭环，而不是一次性整理。

## 隐含假设

- Agent 归纳：高质量知识库不是一次性写出来的，而是靠反复捕获、消化、摄入、查询、再捕获形成。
- Agent 归纳：手动整理仍然有不可替代的认知价值，尤其是在单点知识的拆解、合并和链接阶段。
- Agent 归纳：工具的边界应该按认知动作来划分，而不是按软件功能划分。
- Agent 归纳：Codex 的长期价值来自编排和闭环，而不只是生成内容或代码。

## 还在摇摆的地方

- Agent 归纳：哪些内容应该继续手动 XMind，哪些内容应该直接 cook 成 inbox note，还需要继续形成判断标准。
- Agent 归纳：cook 后的内容何时只是 inbox 暂存，何时应该进入 human raw，再何时值得 ingest/compile，需要更稳定的门槛。
- Agent 归纳：Codex 作为超级入口后，如何避免所有任务都被塞进同一个入口而失去边界，需要靠 workflow contract 约束。

## 可以延伸出的洞察

- Agent 延伸：你的系统正在从“工具集合”变成“agent-centered operating layer”。工具不再各自为政，而是由 Codex 根据任务阶段调度。
- Agent 延伸：XMind 和 Obsidian 的关系不是替代关系，而是粒度关系。XMind 适合局部结构化，Obsidian 适合跨主题链接和长期复用。
- Agent 延伸：cook skills 实际上是在建设一个“知识摄入前端”。它们让外部材料先变成可读、可复盘、可进一步 ingest 的半成品。
- Agent 延伸：真正的闭环不是 `query` 结束，而是 `query` 产生新的捕获、判断、问题或整理动作。

## 可复用原则

- 入口归一，边界分层：Codex 可以是入口，但每类内容仍然要有清楚的归属。
- 手动深加工只保留在高认知收益处：XMind 用在需要亲手拆解和重组的知识点。
- inbox 不是知识库终点：cook note 是待消化材料，不直接等于 canonical source。
- 每个 workflow 都要能回到下一轮：捕获不是终点，查询也不是终点，能进入下一轮才是闭环。
- Agent 归纳和用户原文分离：保留原始想法，避免让系统总结覆盖真实思考。
