## 分析

- 做得好的地方:
  - 父 Agent / worker 并行方式在大规模 ingest 上有效，尤其是明确 write scope、隔离 worktree、按 raw 类型使用 skill。
  - raw-diff、git diff check、wikilink scan 这些简单可复现工具成为可靠门禁。
  - 用户对“抽象太浅、实体稀疏、链接弱”的纠偏推动了 wiki 从 source note 仓库升级为知识图谱。
  - 日报 skill 通过真实使用暴露问题，再反过来修复采集脚本和模板，是一个很好的 workflow 自举案例。
- 做得不好的地方:
  - 初版日报没有按 parent workflow 归并，说明事实采集脚本还停留在任务类型分组，没有真正表达 agent 编排结构。
  - 多个 worker 出现误写原 checkout，说明“worktree 里的 patch 应用路径”仍是高风险点。
  - `render_daily_note.py` 的出现说明 skill 设计边界没有守住：事实抽取可以脚本化，分析和日报正文不能机械生成。
  - read_thread/list_threads 的公开工具能力边界没有先被制度化，导致日报 skill 一开始对公开工具过于理想化。
- 可以沉淀:
  - `collect_session_facts.py` 的 parent workflow、compact、status mapping、tool 摘要能力。
  - 一个 “parent workflow HAT/checklist”：worker 是否有 final message、是否有产物、是否有落盘 diff、是否需要 parent 接管。
  - 一个 worktree worker 写入安全约束：所有 patch 或文件路径必须确认在指定 worktree 下。
- 可以改进:
  - 日报 skill 默认先看 `parent_workflows`，再看 groups，最后才看 individual sessions。
  - 对 worker sessions 统一记录 `agent_role`、`agent_nickname`、`parent_thread_id`、write scope、产物和异常。
  - `read_thread` 使用上明确“最小参数 + cursor 分页 + 失败降级”，不要假设高级参数存在。
