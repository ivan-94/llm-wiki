## Summary

这份 source 把结构化文档提取拆成 Document AI 流水线：布局识别、OCR/文本转换、表格识别、公式识别、图像裁剪、阅读顺序和文档恢复。它同时比较了 LayoutLMv3、DiT、PaddleOCR PP-Structure、Table Transformer、RAGFlow DeepDoc、OmniParser 和 PDF-Extract-Kit 等路线。

## Source Digest

结构化文档提取的目标不是简单 OCR，而是把复杂文档还原为带版面、阅读顺序、表格、公式、图像和语义实体的结构化结果。导图的主流程是先做布局识别，细分文本、表格、公式、图像等区域，再分别用 OCR、表格结构识别、公式识别和图像裁剪提取内容，最后恢复成可阅读、可检索、可供 LLM 消费的文档表示。

模型路线分成三类。第一类是文档图像分类和版面识别，source 记录 BERT 文本模型、DiT 视觉模型、LayoutLMv3/Donut 多模态模型在 RVL-CDIP 上的相对准确率，并强调多模态模型在 Document AI 中更强。第二类是结构化 OCR/版面流水线，以 PaddleOCR PP-StructureV2 为代表，包含方向矫正、布局分析、表格识别、OCR、版面恢复、关键信息提取和结构化输出；其中版面分析涉及 PP-PicoDet、FGD 蒸馏、PubLayNet/CDLA/TableBank/DocBank 等数据集或模型配置。第三类是表格结构识别和 RAG 文档解析，以 Table Transformer、RAGFlow DeepDoc、读光/OmniParser 等作为候选。

多 sheet 内容中，`RAG Flow` sheet 特别强调 PDF 解析复杂性：PDF 文本块具有位置和矩形信息，表格可能来自裁剪图像，图中也可能有标题和文字，因此解析器需要同时处理视觉、OCR、布局和表格结构。`LayoutLMv3` sheet 则把文档理解建模为文本标记、2D 位置信息和图像特征的多模态预训练问题，可针对版面识别等任务再微调。

## Key Claims

- explicit: 结构化文档解析的主流程包括布局识别、文本/OCR 转换、表格识别、公式识别、图像裁剪和文档恢复。
- explicit: source 记录的 RVL-CDIP 对比为：仅文本 BERT 约 89%，仅视觉 DiT 约 92%，LayoutLMv3/Donut 等多模态 Transformer 约 95%。
- explicit: Table Transformer 是类似 DETR 的对象检测模型，在 PubTables-1M 上训练，source 记录其表检测 AP 0.966、表结构识别加功能分析 AP 0.912。
- explicit: PaddleOCR PP-StructureV2 的文档信息提取流程包含图片方向矫正、布局分析、表格识别、OCR、版面恢复和结果恢复成文档。
- explicit: LayoutLMv3 接收文本标记、2D 位置信息和图像特征，并使用 MLM、MIM、WPA 等自监督目标学习文档多模态关系。
- inferred: 面向 RAG 的文档解析应保留布局组件、表格结构和阅读顺序，否则 OCR 文本会丢失文档语义结构。
