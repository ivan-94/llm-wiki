## Summary

这份 XMind 是面向 Agent 执行环境的沙箱方案速记，比较了同机多用户账号、独立容器/虚拟机，以及更轻量的 `nsjail` / `bubblewrap` / `firejail` 路线，并收集了 OpenSandbox、agent-infra/sandbox 等面向 AI Agent 的开源沙箱实现。它的核心关注点是多用户隔离、资源控制、按需创建销毁、运行时工具能力，以及 Agent 如何连接和使用沙箱。

## Source Digest

source 把“沙箱”放在 Agent 基础设施层讨论，而不是只当作单个代码执行工具。最弱的方案是在同一容器或虚拟机中给每个用户创建系统账户和私有目录，它实现简单，适合少量可信团队，但 CPU、内存、磁盘和系统包仍然共享，高负载或逃逸风险会影响其他用户。更强的方案是按用户或会话创建独立容器/虚拟机，换取文件系统、进程空间和资源配额隔离，并可通过 Docker、Kubernetes 或 k3s 做并发调度和生命周期管理。

source 还提示存在比完整容器更轻量的隔离层，点名 `nsjail`、`bubblewrap`、`firejail` 作为比较对象，但未展开细节。开源方案部分把沙箱能力扩展到 Agent 运行时视角：OpenSandbox 强调面向 LLM 应用的命令执行、文件、代码、浏览器和 Agent 运行接口；agent-infra/sandbox 试图把 Browser、Shell、File、MCP 和 VSCode Server 放进单个 Docker 容器；其余项目被列为候选调研对象。整体上，这份 source 适合作为后续整理“Agent 沙箱/执行环境”概念的入口材料。
