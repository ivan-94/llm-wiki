## Summary

这份 XMind 为闪极智能体设计会话持久化与压缩模型。它用 Git 的“工作区 -> 暂存区 -> 仓库”类比，将运行时会话、`messages_staged` 和 `messages` 拆成多阶段存储区域，并定义初始会话、会话恢复和压缩触发时的提交/重置逻辑。

## Source Digest

source 的关键设计是把会话历史拆成可恢复、可压缩、可归档的两层持久化结构。`messages` 类似 Git 仓库，保存已经归档的会话历史；`messages_staged` 类似暂存区，保存尚未提交进 `messages` 的活跃历史；Agent 运行时会话历史类似工作区，每个 turn 结束后持久化到 `messages_staged`，多数情况下它等同于 `messages_staged`。完整历史由 `messages + messages_staged` 组成。

在初始会话中，干净历史直接进入 `messages_staged`；恢复会话时，Agent 直接完整加载 `messages_staged`。当触发压缩时，系统把 `messages_staged` 中除尾部保护区之外的 Summary Area 做摘要并提交到 `messages`，同时忽略 Summary Area 中 summary 消息之前的 Protect Head，因为这部分已经在上一轮压缩中提交过。随后 `messages_staged` 被重置：新的 staged 内容由保护头、压缩后的 summary 和保护尾组成，其中 Protect Head 从第一条 summary 消息之后开始选取。这套模型试图避免重复提交历史，同时保留恢复时的连续上下文和压缩后的可追溯摘要。
