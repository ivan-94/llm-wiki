## Summary

这份 XMind 是一个不侵入业务代码的提示词/LLM 调用评估框架设计。它用 `llm_evaluation/` 独立目录、`INDEX.md` 调用索引、pytest case、OpenAI-compatible provider 配置、LLM-as-a-Judge、results.jsonl 事件流、history 归档、CLI 和 Web UI，构建一个由 agent 驱动的提示词评估闭环。

## Source Digest

该 source 设计的是一个面向项目内 LLM 调用的评估子系统。它的核心原则是“不侵入代码”：通过 `INDEX.md` 扫描并登记项目中的 LLM 调用位置、入口、场景、标识符和评估关注点；评估代码放在独立的 `llm_evaluation/` 目录，不导入项目模块，只把 prompt 抽出来当作可测试对象。目录结构包括 provider/config、AGENTS 指导、domain 场景、prompt.py、pytest 风格 cases.py、results.jsonl、helpers、CLI、FastAPI Web 和 workflow skills。

执行模型基于 pytest 和 OpenAI SDK。评估有 collect 和 gate 两种模式：collect 默认只收集结果，避免中断；gate 在低于阈值时失败，适合 CI。简单 case 是输入、获取 LLM 输出、调用评估器评分；复杂 workflow case 可以组装和 mock 工具调用，并在中间穿插评估。评估器使用更强模型，对 input、result 和 expect 进行 0-10 分评分，并输出 reason；评分标准从 10 分完全满足用户意图、事实准确、格式合规，到 0 分拒答错误、严重幻觉或安全违规。

结果以 append-only 的 results.jsonl 记录为事件流，包含 start、result、end 三类事件。result 事件记录 case 描述、输入、输出、期望、评分、原因、耗时和 usage。每次新评估前可把旧 results.jsonl 归档到 history，文件名包含日期、provider 和 size。CLI 支持 web 和 run：run 接收 provider、size、domain、mode、threshold；web 则提供变量选择、domain 多选、gate/collect、执行中表格化展示、结果筛选、分页详情，以及 domain 专题页的历史列表和耗时、token、评分趋势图。

工作流上，source 设想用 skills 封装 `/index`、`/create`、`/run`：先扫描项目 LLM 调用建立索引，再按 IGNORE/ONLY 和 changelog 生成测试用例，最后运行评估。它还给出一个父 agent fork 多个 sub-agent 并发评估多个 domain、统一模型、提取低分项和汇总改进建议的示例。这使评估系统不只是测试框架，而是 agent 可执行、可并发、可归档、可审阅的提示词质量控制面。
