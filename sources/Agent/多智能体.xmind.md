## Summary

这份 XMind 以多智能体工程编排为主题，先列出 paperclip、ruflo 等方案线索，再重点整理 ClawTeam 和 oh-my-claude-code 两类实现。ClawTeam 代表“CLI + 文件状态 + tmux/subprocess + worktree + 看板/收件箱”的多 agent 团队运行时；oh-my-claude-code 代表“Hooks + Skills + Agents + State”的 Claude Code 编排系统。

## Source Digest

该 source 的核心问题是：多个命令行智能体如何从“并发启动”变成“能分工、通信、跟踪、恢复和验收的团队”。第一张 sheet 只是方案索引，列出 paperclip 和 ruflo 两个候选项目，同时有几个未展开占位分支。后两张 sheet 才是主要内容，分别用 ClawTeam 和 oh-my-claude-code 展示两种多智能体编排模型。

ClawTeam 的模型偏运行时和控制面。它把一个团队拆成 leader、worker、任务板、收件箱、观测面和上下文工具。worker 通常有独立 git worktree、独立 tmux 窗口和独立身份，任务状态通过 pending、in_progress、completed、blocked 以及 blocked-by 依赖表达；通信通过 inbox 的 send、receive、broadcast 实现；观测通过 board show/live、tmux attach 和 Web 面板完成。它不依赖中心化常驻服务，而是把 `~/.clawteam/` 下的 JSON 和目录结构作为真相源，人类和 AI 都通过同一套 CLI 读写状态。驱动方式不是持续轮询 Claude/Codex 内部状态，而是在 spawn 初始提示中注入身份、任务、协作协议和 worker loop，让智能体自己调用 clawteam 子命令。

oh-my-claude-code 的模型偏 Claude Code 内部编排。它用 Hooks 侦测生命周期事件，用 Skills 注入行为，用 Agents 执行专业任务，用 `.omc/` State 在上下文压缩和会话重置后保存进度。其 agent 系统覆盖构建/分析、评审、领域和协调通道；典型流程是 explore、analyst、planner、critic、executor、verifier。Skills 不替换智能体，而是在现有智能体之上增加 autopilot、ralph、ultrawork、team、ccg、ralplan 等工作流能力。状态系统包括 per-mode state、session state、notepad、project memory、plans、notepads、research 和 logs，重点解决长任务和上下文压缩后的连续性。

两者共同说明，多智能体工程的难点不是“有多个 agent”，而是状态外置、任务依赖、通信协议、隔离执行环境、可观测性、计划审批、生命周期管理和上下文恢复。ClawTeam 更像通用 CLI 团队运行时，oh-my-claude-code 更像围绕 Claude Code 的 hook/skill/agent/state 编排层。

## Key Claims

- explicit: ClawTeam 面向多智能体协作和蜂群智能，目标是让多个命令行 AI 编程智能体像团队一样分工、通信和跟进任务。
- explicit: ClawTeam 的 worker 通常有独立 git worktree、独立 tmux 窗口和独立身份，以减少并行改代码冲突。
- explicit: ClawTeam 支持共享任务板、blocked-by 依赖、inbox 消息、终端看板、tmux 平铺和 Web 面板。
- explicit: ClawTeam 以文件系统为真相源，不内置常驻协调服务；状态落在 `~/.clawteam/` 或 `CLAWTEAM_DATA_DIR` 下。
- explicit: ClawTeam 通过 spawn 初始提示注入身份、任务、上下文和协作协议，后续协作靠智能体主动调用 clawteam CLI。
- explicit: oh-my-claude-code 由 Hooks、Skills、Agents、State 四个系统构成，用于生命周期侦测、行为注入、任务执行和进度追踪。
- explicit: oh-my-claude-code 的 `.omc/` 目录保存 state、notepad、project memory、plans、research 和 logs 等持久状态。
- inferred: 多智能体系统的核心控制面包括任务依赖、通信、隔离 worktree、状态持久化、观测和恢复，而不是单纯的 agent 数量。
- inferred: ClawTeam 与 oh-my-claude-code 的差异可概括为“外部 CLI 团队运行时”与“Claude Code 内部 hook/skill 编排层”。
