## Summary

这份 XMind 描述下一代 Agentic Software 的产品形态：软件从固定功能集合转为目标导向系统，UI 从固定界面转为场景化/临时性/个体化界面，核心资产从功能数量转为上下文与记忆，架构从完整 SaaS 转为原子能力加编排层。同时它保留了风险、商业模式、OpenClaw 雏形案例、AI 眼镜作为感知前端，以及传统软件不会立即消失的判断。

## Source Digest

该 source 的中心判断是：Agentic Software 不是在传统软件上加聊天框，而是把软件的入口、资产、架构和商业模式都重新分层。用户给出的不再是精确操作，而是目标和意图；系统负责拆解任务、选择工作流、生成或呈现合适界面，并通过记忆、SOP 和历史偏好形成长期生产力。

资料将 Agentic Software 拆成三层：底层原子能力负责搜索、存储、支付、通信、审批等可复用模块；中间编排层负责任务分解、规则、状态和记忆；上层交互层承接自然语言、动态 UI 和 Agent。与传统 SaaS 的“最佳实践规模化复制”相比，这种形态更适合长尾任务和个体化体验，但也带来不确定性，因此需要计划可见、中间结果可检查、关键步骤可确认、过程可回滚、每一步可验证。

它还把 AI 眼镜放入 Agentic Software 叙事：眼镜不是替代手机，而是作为常时在场、低打断、第一人称、多模态的感知前端；真正的价值需要后台记忆系统把持续感知转换为对用户的理解。该 source 适合编译“Agentic Software”“上下文作为软件资产”“动态 UI”“AI 眼镜作为感知前端”等节点。

## Key Claims

- explicit: 传统软件以按钮、菜单、表单和固定交互路径为主；Agentic 软件接收更模糊的意图，并把意图拆成工作流。
- explicit: Agentic 软件的 UI 可能变成场景化、临时性、个体化界面，同一底层能力会面向不同人、任务和阶段呈现不同界面。
- explicit: 软件壁垒会从功能数量转向上下文、记忆、历史行为、偏好、约定和 SOP。
- explicit: 下一代软件架构可拆为底层原子能力、中间编排层、上层交互层。
- explicit: AI 软件需要审计性和可控性，包括计划可见、中间结果可检查、关键步骤可确认、过程可回滚和逐步验证。
- explicit: 可能出现按成果、按代理席位、按资源消耗、按能力层和按生态分成等商业模式。
- explicit: 传统软件不会整体消失，而会被重新分层、重新定位、重新包裹，成为基础设施。
- inferred: Agentic Software 的关键竞争力会落在“个性化体验 + 可控执行”之间的平衡，而不是单纯模型能力。
- inferred: AI 眼镜若成立，更像 Agentic Software 的感知前端，而不是手机的简单替代品。
