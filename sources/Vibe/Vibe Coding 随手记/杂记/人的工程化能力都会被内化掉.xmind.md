## Summary

这份 XMind 记录了一场围绕“人的工程化能力是否会被 AI/Agent 内化”的长辩论。核心结论是：手动中游干预会减少，很多可观测、调试、优化能力会被 Agent 接管；但系统层面的目标定义、约束、评估和控制不会消失，只会从人手工操作迁移到 Harness/Agent 系统中。

## Source Digest

材料从“工程能力是否消失”开始，先把工程能力重新分层：框架使用者的下限被 AI 工具抬高，能更快生成产品；Harness Builder 负责设计 agent workflow、evaluation、feedback loop、hallucination/cost 控制；系统级设计者负责 problem framing、人机协作结构、哪里用 AI 以及哪里不用。争论的关键不在于普通开发者是否还要写中游实现，而在于中游能力到底是消失、被压缩，还是迁移。

对话逐步细化了任务复杂度边界。对于模型能力高于任务复杂度、任务已被驯化、结构化强、DoD 清晰且可自动验证的场景，上游定义加 AI 执行可以压缩中游，人更多集中在需求定义和结果验收。对于开放复杂域，如多目标 trade-off、长链路决策、模糊目标、高风险系统、强耦合子任务和涌现行为，仅靠前端拆解无法彻底消除复杂度，仍需要运行时调控、约束或至少可观测性。

source 进一步比较了两种控制模型：后验控制是上游定义、AI 黑箱执行、下游发现偏差后再迭代；在线中游控制是在执行过程中持续约束、校验中间结果并动态调整。后验控制适合单次执行成本低、错误代价低、链路短、可快速迭代的场景。复杂系统则需要 trace/logs、行为路径理解和问题定位，否则多轮试错会退化为盲目重试。

最后，材料把“可观测性也会被 Agent 接管”的观点整理为自修复系统：稳定系统需要状态观测、偏差检测和调整机制。当前人类仍必须定义控制目标、约束和评估体系；未来如果 Agent 足够强、问题空间可形式化，手动干预会趋近于消失，但中游控制会转移为 Agent 的能力，而不是从系统中消失。

## Key Claims

- explicit: AI 时代工程能力不是简单消失，而是迁移和分层：框架使用者、Harness Builder、系统级设计者承担不同职责。
- explicit: 绝大多数开发者可能不构建 Harness，开发重心会明显向问题定义和结果评估迁移。
- explicit: 当任务可结构化、可验证、链路短且错误成本低时，上游拆解和后验迭代可以大幅压缩人工中游干预。
- explicit: 开放复杂域中的多目标冲突、强耦合、涌现行为和不可稳定自动验证的 DoD，无法完全通过静态拆解消除。
- explicit: 复杂系统必须有可观测性和行为理解，否则快速迭代会失效。
- explicit: 自修复系统至少需要状态观测、偏差检测和调整机制；人类仍要定义目标、约束和评估体系。
- inferred: “人的工程化能力被内化”更准确地说是工程控制能力从个人手工操作转移到 Agent/Harness 层。
- inferred: 该 source 可支撑区分 Vibe Coding 低成本试错场景与 Agentic Engineering 高约束场景的判断框架。
