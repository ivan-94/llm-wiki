## Summary

这张图把传统软件工程抽象为“共识工程”：在用户/客户、产品/业务、架构师、开发、测试、运维等角色之间，通过需求共识、领域建模、接口共识、架构共识、工程行为共识、可执行共识六层方法，降低协作中的歧义、信息损耗和返工。

## Source Digest

图中将软件交付的关键活动组织为六类共识，每一类都有目标、核心方法论、解决的问题、关键产出和价值。需求共识用 PRD、用户故事、场景和验收标准把“大家解决的是同一个问题”说清楚；领域建模用 DDD、通用语言和限界上下文统一概念体系；接口共识用 API Contract、接口优先设计和契约式设计定义协作边界；架构共识用 ADR、RFC 和架构图记录演化方向与关键决策；工程行为共识用 Code Review、CI/CD 和 Definition of Done 约束多人协作秩序；可执行共识用测试、TDD 和形式化规格把共识转为机器可验证的规范。

这张图的核心价值是把“软件工程实践”重命名为“围绕语义、边界、决策、行为和验证的分层共识系统”。它适合作为后续理解 AI Coding 时代共识对象变化的基线：传统时代的共识主要发生在人与人之间，沉淀物是文档、模型、契约、评审记录和测试。
