## Summary

这份 XMind 提出用持续维护 `implementation-notes.html` 或 Markdown 笔记来控制大型需求中的 AI Coding 偏移。核心不是把 spec 写到没有歧义，而是承认歧义必然存在，并把 agent 在实现中做出的判断、偏差、权衡和未决问题显式化，方便 review 时审计。

## Source Digest

source 将大型需求的偏移控制定位为“判断可审计化”问题：过度规约会让启动变慢且仍无法穷举边界，完全放手则会让隐性决策藏进 diff，等 code review 才暴露。它建议让 agent 在开发过程中同步维护实现说明，记录四类非代码信息：设计决策补足 spec 空白，偏差说明主动偏离规范的原因，权衡记录未采用方案，未决问题标记需要人类回环的点。

这个方法的价值在于降低 agent 的过度澄清倾向，同时避免把所有实现细节推给人类实时跟踪。HTML/Markdown 被视为合适载体，因为它轻量、可读、能随 PR 一起提交。该 source 更像一个执行提示词和审计约束模板，适合后续编译进“AI Coding 偏移控制”“implementation notes 审计”和“大型需求治理”相关概念。
