## Summary

这份思维导图把 Obsidian 解释为一个基于本地 Markdown 文件夹的个人知识管理工具箱：核心是 Vault、Note、Link、Attachment 等文件层对象，上层通过 Markdown、frontmatter、标签、嵌入、Canvas、反向链接、关系图谱、未链接提及和 MOC 形成可连接、可检索、可组织的知识网络。

## Source Digest

source 的主线不是列功能清单，而是建立 Obsidian 的心智模型：它首先是一个本地文件夹中的纯文本知识库，每篇 note 和 attachment 都能被普通文件系统管理；其次是一个以链接为中心的知识网络，双向链接、Graph 和 Unlinked Mentions 负责把分散笔记变成关系；最后是一个类似 Markdown 领域 VSCode 的可扩展工作台，通过命令面板、侧边栏、Canvas 和插件把不同工作流组合起来。

对 AI Wiki 来说，这份 source 能补充“为什么 wiki 使用 Markdown source note、frontmatter、wikilink、MOC/map 和附件策略”的工具基础。它也强调 Obsidian 的价值不来自单一功能，而来自个人如何设计收集、整理、连接和复用笔记的 workflow。
