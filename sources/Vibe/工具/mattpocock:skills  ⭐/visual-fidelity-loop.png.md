## Summary

这张图把 Visual Fidelity Loop 表达为 Target、Discipline、Capture、Diff、Patch、Verify 六步循环，用设计目标、tokens、states、responsive 截图和 evidence 清单证明实现与目标视觉一致。

## Source Digest

图的左侧是 target design 和设计 token/state 输入：颜色、字体、spacing、radius，以及 button/input/toggle/link 的 default、hover、active、disabled、focus 状态。右侧是 browser implementation、mobile/tablet/desktop 响应式截图和 evidence 清单。中间的循环是先定义视觉目标，再应用 token 与规则，捕获实现截图，做放大 diff，修复偏差，最后验证并证明。

它强调视觉还原不能只看单一桌面截图，而要覆盖响应式断点、交互状态、视觉 diff 阈值、回归检查和可审查证据。底部流程把六步压缩为：Target 定义视觉意图；Discipline 应用 token 和规则；Capture 按尺度截图；Diff 检测视觉变化；Patch 修复改进；Verify 验证证明。
