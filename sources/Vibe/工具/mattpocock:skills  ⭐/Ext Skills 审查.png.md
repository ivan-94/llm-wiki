## Summary

这张图把“技能审查员”定义为修复 Agent skill 失败体验的审查角色：修的不是措辞，而是 Agent 实际会失败的地方。

## Source Digest

图的上半部分列出 skill 常见问题：相互冲突、冗余堆叠、困惑歧义、弱引导、目标不清晰、触发不准、边界不明、输出不可用。它强调审查对象不是文本漂亮程度，而是 skill 作为 Agent 操作协议时是否能被正确触发、正确执行并产出可用结果。

图的中部给出 10 个审查视角：触发质量、范围和目标、行为契约、操作路径、冲突和优先级、冗余和上下文成本、歧义和弱引导、组合关系、验证方式、输出可用性。底部把审查结果要求收束为：每个发现都要说明问题、失败模式、修复和替换文本，避免只给泛泛建议。
