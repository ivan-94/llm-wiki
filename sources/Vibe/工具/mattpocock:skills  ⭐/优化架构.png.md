## Source Digest

图中对比了浅模块和深模块：浅模块通常有大接口、薄实现，复杂度散落到调用方，导致理解成本高、改动影响面大、测试越过接口、重复与耦合多；深模块则提供小接口和丰富实现，把复杂度隐藏在 seam 后，通过 adapter 隔离依赖，使理解成本低、改动影响面小、测试聚焦接口、复用与内聚更高。右侧探索信号提醒，当理解一个概念要跳很多文件、接口几乎等于实现、纯函数只为测试而拆、调用方重复处理细节、测试越过接口时，可能存在 deepening opportunity。

探索流程强调先读 CONTEXT.md 和 ADRs，再在代码中寻找 friction，对候选做 deletion test，提交候选清单，由用户选择，再进入 grilling loop；必要时使用 Design It Twice。依赖分类决定测试方式：in-process 可直接合并测试；local-substitutable 用 Fake/Stub；remote owned 定义 port、production adapter 和 in-memory adapter；true external 注入 port 并使用 mock adapter。底部规则将 interface 视为测试面，one adapter 代表假设 seam，two adapters 代表真实 seam，并提示旧浅模块单测可删除，改测深模块接口。
