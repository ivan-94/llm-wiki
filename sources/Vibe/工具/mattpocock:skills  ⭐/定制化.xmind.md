## Source Digest

Workbook 的入口 sheet 把定制化能力分为 HAT、Agent Runtime、PR 合并、流程控制、实现交付、PR 创建、交叉 review、上下文检查、PRD 批量交付和人机验收。真正展开最多的是 HAT 和上下文审计：`/hat-prepare` 负责为用户视角完整路径准备验收环境、验收清单、数据需求和幂等 `prepare.sh`，并区分 blank/fork/attach 三种环境模式；`/hat-run` 只读取 `/hat-prepare` 产物，调用准备脚本，按 P0/P1/P2 执行 API、页面、数据库、CLI、日志等验收，并输出 summary、manual 标记、截图和日志。

HAT 相关 sheet 还记录了现实阻碍：验证码/认证、复杂前端组件、服务依赖生命周期、权限与验收数据不足。对应策略不是让 Agent 硬猜 UI，而是准备可验收账号/token、优先 API 验证、给组件稳定入口、明确服务启停与恢复检查，并为项目写 `HAT.md` 作为通用记忆层。`/hat-frontend-friendly` 进一步把稳定 testid/aria、URL 化状态、关键链路日志和 Playwright 测试专用动作列为原则。

流程治理部分说明 `/setup-agent-workflow` 要把离散 skills 串成流程，并通过 AGENTS.md 全局引导和规范化来源引用解决长流程中的 handoff 损耗。`/deliver-issue`、`/create-pr`、`/cross-review`、`/deliver-prd` 等 sheet 大多是占位或简述，但表达了从实现、review、HAT 到 PR 的流水线关系。`/agent-context-audit` 则最完整：它把 sub-agent 当答题者、主 agent 当考官，用独立场景题、二阶段追问和 ground truth 对比，诊断上下文污染、指令冲突、探索成本和文档入口问题。
