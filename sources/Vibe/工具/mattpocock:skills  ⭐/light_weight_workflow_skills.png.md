## Summary

这张图把轻量 workflow skills 归为沟通、思考、写作和知识管理工具：`caveman`、`grill-me`、`write-a-skill`、`edit-article`、`obsidian-vault`、`zoom-out`。它强调短 skill 也要有清楚触发词、持续性、输出形态和停止条件。

## Source Digest

图上方说明这类 skill 信息量不大但触发频繁，会改变对话方式、提问方式、写作方式和知识组织方式。`caveman` 是极简沟通模式，规则包括删冠词/客套/填充词、保留技术准确性、短句和主动语态，触发后持续到用户说 stop caveman。`grill-me` 用连续访谈逼近清晰决策，原则是一次只问一个问题、每题给推荐答案、能查代码就查代码、直到决策树分支被解决。`write-a-skill` 用于创建可复用流程，强调先收集需求与边界，再草拟 `SKILL.md`，超过约 100 行则拆 reference，确定性操作可加 scripts。`edit-article` 用分 section、DAG 信息依赖和逐步重写来改善文章。`obsidian-vault` 管理 Obsidian 笔记，强调稳定 title case、浅层目录、index notes、wikilinks 和 backlinks。`zoom-out` 在不熟悉代码区域时上升一层抽象，识别相关模块、调用者与项目 domain glossary。

## Key Claims

- explicit: 轻量 workflow skills 需要明确触发词、持续性、输出形态和停止条件。

- explicit: `grill-me` 面向设计或计划的连续访谈，核心是一次只问一个问题并推动决策树收敛。

- inferred: 这些 skill 不主要改变代码产物，而是把高频协作方式封装成可触发的行为协议。
