## Summary

这份 XMind 聚焦 Vibe/agentic coding 的实战上下文工程：上下文拥有权、频繁有意压缩、token budget 管理、规格驱动流程、上下文隔离，以及在研究和设计阶段引入高杠杆人工审查。

## Source Digest

source 把 HumanLayer 的 advanced-context-engineering 思路转译为编程 agent 工作流：agent 不应被动吞下所有上下文，而应主动管理它看见的上下文，选择、过滤、压缩并隔离片段。上下文不是越多越好，应在 research、planning、implement 等阶段持续精炼，完成子任务后提炼输出并剥离冗余，下一阶段只加载需要的局部信息；当输出开始偏离时，可以重置上下文。

它还提出 token budget ownership：让上下文窗口保持在足够信息但不过载的区间，材料中给出 40-60% 作为经验目标。规格驱动 workflow 将任务拆成研究、计划/规格、实现三段：研究阶段发现代码结构、模块依赖、历史记录和相关文档；计划阶段产出方案、接口、架构、模块和流程设计；实现阶段用设计文档和局部子上下文驱动编码、测试和整合。

最后，source 强调隔离噪声与高杠杆人工审查。不同任务可用 sub-agent 或上下文分片保持干净环境；人工介入越靠前越有杠杆，尤其是在研究和设计阶段审查设计文档，能以较低成本纠正方向，避免代码完成后再大规模返工。

## Key Claims

- explicit: 编程 agent 应主动拥有和管理上下文，而不是盲目接受全部上下文。
- explicit: 开发流程中应频繁、有意地压缩上下文，按阶段去冗余、分层并聚焦下一步需要的信息。
- explicit: 上下文窗口需要预算管理，source 给出 40-60% 填充度作为经验目标。
- explicit: 规格驱动流程应遵循研究、计划/规格、实现的阶段划分，并让中间产物成为下一阶段输入。
- explicit: 人工审查在研究和设计阶段杠杆更高，可降低实现后返工成本。
- inferred: 这份 source 适合补充“上下文工程”概念中 agentic coding 的操作层方法。
