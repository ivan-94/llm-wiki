## Summary

这份 XMind 是一份大模型入门通识，覆盖 LLM 类型、推理流程、输出优化、量化、评估、采样超参、训练数据形态、tokenizer/chat template、loss、SFT/LoRA/QLoRA 和 PyTorch 基础，并包含一个基于 ModelScope 加载 Llama 3 Instruct 的推理示例。

## Source Digest

这份资料把“大模型”拆成两个互补视角：先从使用者视角说明 base、chat、多模态、agent、code 等模型类型，以及 prompt、few-shot、RAG、微调、量化、评估这些常见改进路径；再从训练者视角解释模型、训练、推理、数据、tokenizer、loss 和微调参数。它的核心学习价值是把 LLM 的“外部能力分类”和“内部训练/推理机制”连起来，帮助读者判断一个问题应该用提示语、检索、微调、量化还是评估体系来处理。

高信号细节包括：base 模型只是预测后续文本，chat 模型是在 base 上通过指令/对话数据和对齐训练获得助手式响应；agent 能力不是单个模型类型，而是模型与规划、记忆、工具调用等组件协作后的系统能力；code 模型来自预训练和 SFT 阶段更高比例的代码数据。推理流程被压缩为 tokenizer 分词、embedding、Transformer 层计算、logits、采样和 EOS 停止。训练部分用 `y = ax + b` 到 `y = sigma(ax+b)` 的类比解释拟合，用未标注数据、已标注问答数据、带正误偏好数据区分预训练、微调和对齐。

实践决策层面，这份资料给出几个场景分流：从零训练适合大量算力和刷榜；领域知识未包含在预训练语料中可继续训练；有标注数据和任务范式时可微调；回答必须严格基于已有知识时更适合 RAG 或微调后结合 RAG；希望问答机器人满足特定范式时考虑微调加对齐。微调细节上，它区分全量训练和轻量训练，强调 LoRA/QLoRA、量化、batch size、learning rate、max length、flash attention、gradient checkpointing 等参数和组件对显存、速度、稳定性和能力保持的影响。

## Key Claims

- explicit: base 模型是在海量文本上训练出来的续写模型，后续文本不一定是对指令或对话的响应。
- explicit: chat 模型是在 base 模型基础上通过指令-响应数据和对齐训练得到的，更符合人类对助手响应的预期。
- explicit: LLM 生成过程包括分词、embedding、Transformer 层计算、logits、采样和重复生成直到目标长度或 EOS。
- explicit: 微调会改变模型权重，可以更好控制生成结果，但可能带来意外行为或通用能力下降，因此需要客观评估。
- explicit: 量化用低精度表示权重、偏置或激活，可减少内存和计算需求，但需要在效率和精度之间取舍。
- explicit: 训练数据形态影响训练阶段；未标注数据常用于预训练，标注问答数据可用于微调，带正确/错误答案的数据可用于对齐。
- explicit: tokenizer 不等于按字切分；不同模型有自己的 tokenizer 和 chat template，开源模型之间没有统一 template 标准。
- explicit: LoRA 是轻量微调的流行结构，QLoRA 可理解为量化加 LoRA。
- inferred: 这份资料更适合作为 LLM 基础学习地图的总入口，而不是某个单一技术的深入页。
- inferred: 资料中的场景分流可编译为“提示语/RAG/微调/继续训练/对齐训练”的选型决策框架。
