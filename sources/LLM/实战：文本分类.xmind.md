## Summary

这份 XMind 是一个较稀疏的文本分类实战提纲，核心观点是 Transformer 表示模型和生成模型都可用于文本分类；表示模型路径包括直接使用或微调分类模型，以及用输入文本和标签 embedding 的余弦相似度完成分类。

## Source Digest

资料把文本分类分成“表示模型”和“生成模型”两条路线。表示模型路线更具体：一类是直接微调或使用已经面向特定标签空间训练好的模型，例如情感分类模型；另一类是把输入内容和候选标签分别向量化，再用余弦相似度选择最接近的标签。基础模型选择上，资料指出 BERT 是常见的 encoder-only 模型，并存在很多变体。

生成模型路线在 workbook 中只保留了标题，没有展开 prompt 设计、输出解析、few-shot、评估或成本延迟等细节。因此这份 source 更像是文本分类任务选型的骨架：当标签体系稳定、追求低延迟和可解释 embedding 相似度时，可优先考虑表示模型；当分类规则更开放或需要自然语言推理时，生成模型可能是候选，但需要后续资料补全。
