## Summary

这份导图围绕模型微调的动机、与提示词工程的对比、预训练/微调过程、指令微调、数据准备、训练、评估和工具生态展开。它把微调定义为把更多任务或领域数据放入模型训练过程，使预训练模型适配特定知识、风格和行为。

## Source Digest

导图首先回答“为什么要微调”：让模型获取专业领域知识、形成一致风格、减少幻觉，并通过 GPT 到 ChatGPT、GPT 到 GitHub Copilot 的例子说明通用模型可被进一步调整成特定产品或任务形态。微调被描述为把更多数据放入模型中，让模型学习这些数据，从而在专业用途、输出风格和可靠性上改进。

与提示词工程的对比部分把微调的价值归为四组：性能、私有数据、成本和可靠性。性能侧包括减少幻觉、增加一致性、提高内容质量、减少不想要的信息；私有数据侧包括隐私保护、避免数据泄露和学习新知识；成本侧强调更低成本、透明性和可控性；可靠性侧强调低延迟、审查和运行控制。这里的分类有些混杂，但反映了作者关注的不只是准确率，也包括企业应用中的安全、成本、延迟和治理。

训练过程部分区分预训练和微调。预训练是在大规模无标签语料上进行的昂贵初始化训练，目标常被表述为预测下一个 token，数据通常来自清洗后的网络语料，闭源模型数据集不公开，开源数据集例子包括 The Pile 和 Hugging Face 上的数据集。预训练阶段让模型学习通用语言规律，但基础模型仍更像 next-token predictor，需要后续微调才能更好地响应任务。

微调部分强调在预训练模型上使用特定任务数据进一步训练，通常采用有监督学习，使用较小数据集，目标是学习任务特定特征、提高准确性并适配场景。导图还记录了两类微调路径：一种是“借花献佛”，即选择大模型擅长的任务，规划约 1000 个输入，用大模型生成输出并人工优化，再用这些数据微调小模型；另一种是指令微调，把指令或提示与任务输入一起提供，使模型更像 chatbot，更适合大众交互，Alpaca 被作为基于 ChatGPT 生成 Q&A 数据的例子。

主要步骤部分保留了数据准备、训练和评估的框架。数据准备强调高质量、多样、真实、大量的指令-响应对，合并到提示模板中，分词、填充、对齐，并拆分训练集和测试集；分词被解释为把 token 转为数字表示，不同模型使用的编码器/解码器不同。评估部分列出人类评估、测试集评估，以及 ARC、HellaSwag、MMLU、TruthfulQA、Elo 排名等测试维度。工具部分列出 PyTorch、Hugging Face 和 Llama library/Lamini。
