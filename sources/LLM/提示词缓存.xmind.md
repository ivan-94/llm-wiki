## Source Digest

资料把 prompt caching 视为多供应商 API 能力差异，而不是单一模型技巧。它记录了几个可比较维度：缓存是自动触发、隐式支持还是显式创建；显式缓存是否接近 Anthropic 的 checkpoint/cache-control 风格；是否只对自家模型可用；是否需要额外维护 context/cache API；代理层 LiteLLM 是否能自动注入缓存 checkpoint。

从供应商差异看，OpenAI 被记录为自动缓存；Anthropic 支持自动缓存和显式缓存；Gemini 支持显式和隐式，但显式需要手动创建；阿里百炼支持隐式和显式，显式体验接近 Anthropic；火山的自动缓存只支持自家模型，显式缓存对 Response API 更友好，而 Context API 需要手动维护；LiteLLM 被列为扩展阅读，用于 prompt caching 和自动注入缓存检查点。所有外部链接均来自 raw，未联网核验。
