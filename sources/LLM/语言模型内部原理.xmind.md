## Summary

## Source Digest

资料从外部表现切入：语言模型一次生成一个 token，并根据前文预测后文，因此属于自回归模型。随后它拆解内部结构：分词器负责把文本变成 token；LM Head 把隐藏状态映射到词表概率；解码策略决定如何从概率分布中选择下一个 token，greedy decoding 是最简单的最高概率选择，但容易无趣。

Transformer 部分把输入看作并行 token stream，每个 token 输入一个向量并输出一个向量，上下文大小对应可并行处理的 token 数量。KV cache 的意义是复用此前 token 序列的 key/value 计算结果，避免生成后续 token 时重复计算。Transformer block 被拆成注意力层和前馈神经网络：注意力层负责让当前位置结合前序 token 的相关信息，多头注意力让模型同时学习不同关注模式；前馈网络负责复杂非线性变换，是模型参数和训练后信息的重要承载处。

注意力机制的高信号细节包括：当前位置的 query 向量表达“我要找什么信息”，key 向量表达其他 token 可被匹配的标识，value 向量是最终被聚合的信息来源；query 与 key 的点积产生相关性分数，softmax 归一化后作为权重，对 value 向量加权求和，形成当前位置更具上下文意义的新表示。

## Key Claims

- explicit: 语言模型一次生成一个 token，根据前面内容预测后面内容，这被称为自回归模型。
- explicit: LM Head 用于把模型隐藏状态映射到词汇表大小的输出空间，从而预测下一个 token 的概率分布。
- explicit: 解码策略决定如何从概率列表中选择结果；greedy decoding 选择概率最高的结果，但通常会较无趣。
- explicit: Transformer 架构的重要优势是并行化，每个输入 token 都是一个输入向量到输出向量的 stream。
- explicit: KV cache 的动机是在生成后续 token 时复用此前序列的计算结果。
- explicit: Transformer block 包含注意力层和前馈神经网络两个重要组件。
- explicit: 多头注意力通过并行复制注意力机制，让模型同时关注输入数据的不同部分和不同模式。
- explicit: query、key、value 三种投影矩阵通过线性变换帮助模型计算注意力分数和加权求和。
- explicit: 前馈神经网络提供复杂非线性变换能力，训练之后的大部分信息保存在这里。
