## Source Digest

该 source 是一个垂直场景的提示语优化框架。它没有给出完整 prompt，而是列出设计公文生成/优化提示语前必须澄清的变量。

题材分支强调公文不是单一任务。通知、会议纪要、人大议案等类型有不同的文体目的、行文结构和格式规范。例如议案可能需要案由、建议等结构；通知和会议纪要则分别偏向事项传达和会议记录整理。

案例分支要求分别模拟输入和输出。输入侧要明确不同题材下用户会提供哪些信息，输出侧要定义期望效果。这对应提示语工程中的“任务输入契约”和“输出验收标准”。

选题内容分支提出标题、目录、主要思想等内容变量，说明 prompt 需要先确定文档要表达什么，再规定怎么写。

测试分支从篇幅和模型对比两个维度评估 prompt：短篇幅与中长复杂篇幅对模型的结构保持能力要求不同；国内外模型、模型规模和指令遵循度会影响同一 prompt 的表现。

这张图的价值在于把通用提示语工程落到“公文”场景：先拆文体和输入输出契约，再用多模型、多篇幅测试验证 prompt 是否稳定。

## Key Claims

- 垂直场景提示语优化必须先识别任务类型和文体结构，不能只写通用生成指令。
- 公文生成 prompt 需要明确输入字段、输出格式、行文结构、排版格式和验收标准。
- prompt 测试应覆盖不同篇幅和复杂度，避免只对短样例有效。
- 模型对比应关注输出指令遵循度、规模差异和国内外模型表现。
