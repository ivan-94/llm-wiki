## Summary

这个 workbook 是提示语工程的多 sheet 学习体系，覆盖基础、结构框架、原则/通用技巧、常规高级技巧、故障排除、Agent 模式、长上下文模型、其他高级技巧、Agent 工程 12 法则和 FAQ。

## Source Digest

该 source 不只是目录，而是把提示语工程拆成从基础到 Agent 工程化的学习路径。

`__init__` sheet 是总入口，列出提示语库和学习来源。它把 Claude Prompt Library、OpenAI Prompt Engineering、Anthropic chain prompts、Prompting Guide、Learn Prompting、modern advances 等作为外部学习来源。

`基础` sheet 强调提示工程是一门经验科学：大部分工作不是一次性写 prompt，而是建立评估、测试和迭代循环。它还整理了常见采样/生成参数，如 temperature、top_p、max length、stop sequences、frequency penalty、presence penalty，并把角色、zero-shot、few-shot、提示要素、提示语生成器作为基础模块。

`原则/通用技巧` sheet 关注可迁移原则：指令要清楚，避免让模型猜；角色提示能改变语气、准确性和上下文；few-shot 示例应代表期望输入输出、推理过程和行为风格；复杂任务应拆成子任务；长对话需要总结和过滤历史；必要时应使用外部工具补偿模型弱点；预填充回复和结构化输出可以提高格式稳定性；反思/重写可以作为提示迭代的一部分。

`提示语结构框架` sheet 把 prompt 看成结构化任务说明，常见框架包括基础框架、CRISPE、Co-star、Camel 等。它们差异不大，核心都是表达角色、目标、上下文、风格、语气、受众和响应格式。

`常规高级技巧` sheet 整理高级 prompting 技术，包括 zero-shot、few-shot/in-context learning、Chain-of-Thought、Zero-shot CoT、Auto-CoT、Self-Consistency、Tree of Thoughts、PAL、Least-to-Most、Prompt Chaining、Generated Knowledge Prompting、RAG 和 ReAct。这些技巧共同说明：提示语工程不仅是“写清楚”，还包括构造推理路径、检索知识、调用工具和分解任务。

`故障排除` sheet 把提示语工程和运行质量联系起来：减少延迟可以从模型选择、prompt/输出长度、temperature、stream 等方向优化；幻觉问题需要定义、策略和参考文献；风险和安全包括越狱/提示注入、提示泄露、角色外反应和非确定性输出。

`Agent 模式`、`长上下文模型`、`其他高级技巧` 和 `12 法则` 将提示语工程推进到 Agent 工程。它们强调自然语言到工具调用、prompt 资产版本化、上下文窗口自主管理、工具调用作为结构化输出、执行状态和业务状态统一、启动/暂停/恢复 API、人类沟通工具化、显式控制流、错误压缩进上下文、小而聚焦的代理、无状态 reducer、从任意地方触发并到达用户所在工作流。

它与 `上下文工程.xmind` 的关系是：提示语工程关注“如何表达任务和约束”，上下文工程关注“模型执行时可见的信息环境”。在 Agent 场景里，两者会合流，因为系统提示、工具 schema、工具结果、状态摘要、检索内容、错误摘要和历史压缩都会一起构成模型上下文。

## Key Claims

- 提示语工程可以从基础概念、结构框架、通用技巧、高级推理技巧、故障排除、Agent 模式和长上下文几个维度系统学习。
- 高质量提示语库适合用作模仿对象，帮助识别成熟 prompt 的组成部分。
- Agent 模式和长上下文模型把提示语工程推向更复杂的上下文管理问题。
- 提示工程是经验科学，核心循环是评估、测试和迭代，而不是一次性写出完美 prompt。
- 高级技巧可以按推理增强、任务分解、知识检索、工具调用和结构化输出来归类。
- Agent 工程中的 prompt 要版本化、可测试，并和工具 schema、控制流、状态管理一起设计。
